한국관광공사(KNTO)와 한국사회보장정보원(SSIS)의 외부 API 데이터를 수집, 정제, 적재(ETL)하여 내부 시스템의 `PLACE` 도메인에 통합하기 위한 기술적 결정 사항 및 아키텍처를 기록합니다.

------

## 1. 개요 및 기술 스택

- **목적**: 이기종 공공데이터 API를 병합하고, 유동적인 자연어 무장애 텍스트 데이터를 정형화하여 내부 서비스에서 검색 및 렌더링이 용이하도록 구성합니다.
- **기술 스택**: Java Spring Batch, PostgreSQL, PostGIS (공간 처리)

------

## 2. ETL 파이프라인 설계

파이프라인은 두 API의 특성을 고려하여 각각 별도의 Batch Job으로 분리 운영합니다.

### 2.1. Extract (데이터 수집 계층)

- 동기화 전략

  : 

  증분 수집

  - 한국관광공사의 `areaBasedSyncList2` API를 활용하여 마지막 동기화 시점(`modifiedtime`) 이후 변경 및 삭제(`showflag`)된 데이터만 호출합니다.

- **배치 주기**: 매주 일요일 23:59

### 2.2. Transform (정제 및 변환 계층)

수집된 원시 데이터를 내부 DTO 및 도메인에 맞게 규격화합니다.

1. **공간 규격화**: 서로 다른 좌표계 변수명(`mapx/mapy`, `faclLng/faclLat`)을 PostGIS `Point` 타입으로 변환합니다.

2. 텍스트 구조화 파싱 (Hybrid Approach)

   :

   - **Rule-based (1차)**: 정해진 패턴에 대해 정규식 및 키워드 사전을 적용하여 JSON 형태로 구조화합니다.
   - **LLM Fallback (2차)**: 정해진 패턴을 벗어난 예외적인 자유 텍스트의 경우, 내부적으로 경량 LLM API를 호출해 JSON 구조로 강제 파싱합니다.

3. 데이터 중복 검사 (Deduplication)

   :

   - *보완용인 사회보장정보원(SSIS) 데이터를 적재할 때 수행합니다.*
   - PostGIS의 `ST_DWithin`을 활용하여 반경 50m 내에 등록된 장소가 있는지 공간 쿼리를 실행합니다.
   - `pg_trgm` 모듈을 이용해 장소명 유사도를 검증하고, 일치할 경우 기존 `PLACE`의 메타데이터에 속성을 병합합니다. 일치하지 않으면 새 장소로 추가합니다.

### 2.3. Load (DB 적재 계층)

- 적재 전략

  : 

  Chunk 기반 Upsert (`ON CONFLICT`)

  - Spring Batch의 `JdbcBatchItemWriter`를 활용하여 1,000건 단위의 Bulk Upsert를 실행합니다.
  - 매주 수천 건 내외의 증분 데이터만 처리되므로 쓰기 부하는 극히 낮습니다.

------

## 3. Spring Batch 구현 계획

- **Extract (`ItemReader`)**: `RestClient`를 활용한 외부 API 페이징 호출 및 증분 수집 (버퍼링 방식 적용).

- **Transform (`ItemProcessor`)**:

  1. 정규식 + LLM 하이브리드 파싱 (자연어 -> JSONB)
  2. PostGIS 및 `pg_trgm` 기반의 공간/문자열 중복 검증 (SSIS 데이터 보완용)

- **Load (`ItemWriter`)**: `JdbcBatchItemWriter`를 활용한 고성능 벌크 Upsert (`ON CONFLICT`).

- 1. Extract 모듈

  API의 페이징(PageNo) 처리를 위해 `AbstractItemCountingItemStreamItemReader`를 상속받고, Spring Boot 3.2의 `RestClient`를 사용하여 동기식 API 호출을 수행합니다.

  ```java
  public class KntoApiItemReader extends AbstractItemCountingItemStreamItemReader<PlaceDto> {
  
      private final RestClient restClient;
      private final String serviceKey;
      private final String lastSyncTime; // 증분 수집 파라미터 (예: "20260614")
      
      private int currentPage = 1;
      private final int numOfRows = 1000;
      private Queue<PlaceDto> itemBuffer = new LinkedList<>();
  
      public KntoApiItemReader(RestClient restClient, String serviceKey, String lastSyncTime) {
          this.restClient = restClient;
          this.serviceKey = serviceKey;
          this.lastSyncTime = lastSyncTime;
          setName("KntoApiItemReader");
      }
  
      @Override
      protected PlaceDto doRead() throws Exception {
          if (itemBuffer.isEmpty()) {
              fetchNextPage();
          }
          // 버퍼가 비었으면 null을 반환하여 Reader 종료 (Chunk 끝)
          return itemBuffer.poll();
      }
  
      private void fetchNextPage() {
          KntoApiResponse response = restClient.get()
              .uri(uriBuilder -> uriBuilder
                  .scheme("http")
                  .host("apis.data.go.kr")
                  .path("/B551011/KorWithService2/areaBasedSyncList2")
                  .queryParam("serviceKey", serviceKey)
                  .queryParam("modifiedtime", lastSyncTime)
                  .queryParam("numOfRows", numOfRows)
                  .queryParam("pageNo", currentPage)
                  .queryParam("MobileOS", "ETC")
                  .queryParam("MobileApp", "App")
                  .queryParam("_type", "json")
                  .build())
              .retrieve()
              .body(KntoApiResponse.class);
  
          List<PlaceDto> items = response.extractItems();
          if (items != null && !items.isEmpty()) {
              itemBuffer.addAll(items);
              currentPage++;
          }
      }
  
      @Override protected void doOpen() throws Exception {}
      @Override protected void doClose() throws Exception {}
  }
  ```

- 1. Transform 모듈

  ### 1) JPA Repository (공간 쿼리)

  PostGIS의 `ST_DWithin`과 `pg_trgm`의 `similarity` 함수를 활용한 JPA Native Query입니다.

  ```java
  import org.springframework.data.jpa.repository.JpaRepository;
  import org.springframework.data.jpa.repository.Query;
  import org.springframework.data.repository.query.Param;
  import java.util.Optional;
  
  public interface PlaceRepository extends JpaRepository<Place, Long> {
      @Query(value = """
          SELECT * FROM place p
          WHERE ST_DWithin(p.location_point, ST_SetSRID(ST_MakePoint(:lng, :lat), 4326), 50, false)
          AND similarity(p.name, :name) > 0.5
          ORDER BY similarity(p.name, :name) DESC
          LIMIT 1
          """, nativeQuery = true)
      Optional<Place> findDuplicateFacility(@Param("lng") double lng, @Param("lat") double lat, @Param("name") String name);
  }
  ```

  ### 2) 무장애 텍스트 하이브리드 파서

  정규식을 1차로 시도하고, 실패한 미지의 텍스트만 LLM API를 호출하여 비용을 절감합니다.

  ```java
  import org.springframework.batch.item.ItemProcessor;
  import org.springframework.stereotype.Component;
  import lombok.RequiredArgsConstructor;
  import java.util.regex.Matcher;
  import java.util.regex.Pattern;
  
  @Component
  @RequiredArgsConstructor
  public class BarrierFreeParsingProcessor implements ItemProcessor<PlaceDto, PlaceDto> {
  
      private final LlmService llmService;
      private static final Pattern WHEELCHAIR_PATTERN = Pattern.compile("(대여가능|있음|보유).*?휠체어.*?(\\d+)대?");
  
      @Override
      public PlaceDto process(PlaceDto item) {
          String rawText = item.getRawWheelchairText();
          BfDetailJson parsedDetail = parseByRegex(rawText);
  
          // 정규식 실패 시 예외적으로 LLM(Fallback) 호출
          if (parsedDetail == null && rawText != null && !rawText.isBlank()) {
              parsedDetail = llmService.parseToStructuredJson(rawText, "WHEELCHAIR");
          }
  
          if (parsedDetail != null) {
              item.setWheelchairDetail(parsedDetail);
          }
          return item;
      }
  
      private BfDetailJson parseByRegex(String rawText) {
          if (rawText == null || rawText.isBlank()) return null;
          Matcher matcher = WHEELCHAIR_PATTERN.matcher(rawText);
          if (matcher.find()) {
              Integer count = (matcher.groupCount() >= 2 && matcher.group(2) != null)
                              ? Integer.parseInt(matcher.group(2)) : null;
              return new BfDetailJson(true, count, rawText);
          }
          return null;
      }
  }
  ```

  ### 3) CompositeItemProcessor 구성

  위의 파서와 JPA 중복검증 로직을 하나의 체인으로 묶습니다.

  ```java
  @Bean
  public ItemProcessor<PlaceDto, PlaceDto> compositePlaceProcessor(
          BarrierFreeParsingProcessor parsingProcessor,
          SpatialDeduplicationProcessor deduplicationProcessor) {
  
      CompositeItemProcessor<PlaceDto, PlaceDto> compositeProcessor = new CompositeItemProcessor<>();
      compositeProcessor.setDelegates(List.of(parsingProcessor, deduplicationProcessor));
      return compositeProcessor;
  }
  ```

- 1. Load 모듈

  - 여기서는 메모리 효율성을 위해서 JPA 영속성 컨텍스트를 타지 않고 직접 네이티브 쿼리를 날립니다.

  ```java
  import org.springframework.batch.item.database.JdbcBatchItemWriter;
  import org.springframework.batch.item.database.builder.JdbcBatchItemWriterBuilder;
  import org.springframework.context.annotation.Bean;
  import org.springframework.context.annotation.Configuration;
  import javax.sql.DataSource;
  
  @Configuration
  public class PlaceItemWriterConfig {
  
      @Bean
      public JdbcBatchItemWriter<PlaceDto> placeUpsertWriter(DataSource dataSource) {
  
          String upsertSql = """
              INSERT INTO place (external_id, source, name, location_point)
              VALUES (:externalId, :source, :name, ST_SetSRID(ST_MakePoint(:lng, :lat), 4326))
              ON CONFLICT (external_id, source)
              DO UPDATE SET
                  name = EXCLUDED.name,
                  location_point = EXCLUDED.location_point
              """;
  
          return new JdbcBatchItemWriterBuilder<PlaceDto>()
                  .dataSource(dataSource)
                  .sql(upsertSql)
                  .beanMapped()
                  .assertUpdates(false)
                  .build();
      }
  }
  ```

- 최종 Job Step

  ```java
  @Bean
  public Step kntoSyncStep(JobRepository jobRepository, PlatformTransactionManager transactionManager) {
      return new StepBuilder("kntoSyncStep", jobRepository)
              .<PlaceDto, PlaceDto>chunk(1000, transactionManager) // 1,000건씩 Bulk 트랜잭션
              .reader(new KntoApiItemReader(restClient, serviceKey, lastSyncTime))
              .processor(compositePlaceProcessor)
              .writer(placeUpsertWriter)
              .build();
  }
  ```
